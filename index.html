CSS 3D transform变换那些事

3D transform有下面三个方法：

  • rotateX(angle)
  • rotateY(angle)
  • rotateZ(angle)

XYZ轴就像下面这张图一样：

简而言之，X轴是向前扑倒，Y轴是原地转圈，Z轴是平躺着

必不可少的perspective属性

　　
这个属性是干嘛的？简单来说没有这个属性我们看到的东西都是2D，不会有3D的效果。
　　
perspective的中文意思是：透视，视角！
　　
perspective属性的存在与否决定了你所看到的是2次元的还是3次元的，也就是是2D transform还是3D transform. 这不难理解，没有透视，不成3D。
　　
CSS3 3D transform的透视点是在浏览器的前方，大概就是我们眼睛的方位。
　　
为了更好的观察和理解perspective属性，我们用translateZ属性来说明,translateZ表示的是元素在Z轴移动的距离，换句话说，translateZ的功能就是让元素在自己的眼前或近或远。比方说，我们设置元素perspective为201像素，如下：

则其子元素，设置的translateZ值越小，则子元素大小越小（因为元素远去，我们眼睛看到的就会变小）；translateZ值越大，该元素也会越来越大，当translateZ值非常接近201像素，但是不超过201像素的时候（如200像素），该元素的大小就会撑满整个屏幕（如果父辈元素没有类似overflow:hidden的限制的话）。因为这个时候，子元素正好移到了你的眼睛前面，所谓“一叶蔽目，不见泰山”，就是这么回事。当translateZ值再变大，超过201像素的时候，该元素看不见了——这很好理解：我们是看不见眼睛后面的东西的！

透视角度perspective-origin属性

　　
这个属性的意思是眼睛所看的位置。
　　
这个属性有两个参数：perspective-origin: X Y ;，默认为舞台中央即50% 50%，前一个属性代表向X轴的偏移量，后一个代表Y轴的偏移量。

transform-style:preserve-3d

transform-style属性也是3D效果中经常使用的，其两个参数，flat|preserve-3d. 前者flat为默认值，表示平面的；后者preserve-3d表示3D透视。
　
基本上，我们想要根据现实经验实现一些3D效果的时候，transform-style: preserve-3d是少不了的。一般而言，该声明应用在3D变换的兄弟元素们的父元素上，也就是舞台元素。

事件

事件流

事件流描述的是从页面中接受事件的顺序。IE下的事件流是事件冒泡流，标准浏览器下是事件捕获流。

  • 事件冒泡流（IE）

事件开始时有最具体的元素，逐级向上传播到较为不具体的节点（文档）。

如果你点击了dome中的div元素，那么这个click事件会按照下面如下顺序传播：

  1. <div>
  2. <body>
  3. <html>
  4. document

也就是说click事件首先在

元素上发生，然后，click事件沿DOM树向上传播，在每一级节点上都会发生，直至传播到document对象。

所有浏览器都支持事件冒泡

  • 事件捕获流（Netscape）

事件捕获的思想是不太具体的节点应该更早接收到事件，而最具体的节点应该最后接收到事件。

事件捕获的真实用意在于在事件到达预定目的地之前捕获它。

以上面的例子举例，单击<div>元素会以下列顺序触发click事件：

  1. document
  2. <html>
  3. <body>
  4. <div>

事件捕获过程中document对象首先接收到click事件，然后沿着DOM树向下，一直传播到事件的实际目标，即<div>元素。

IE9（IE8及以下不支持），Safari,Chorme,Opera,Firefox都支持事件捕获。

DOM事件流

DOM2级事件规定的事件流包括三个阶段：事件捕获阶段、处于目标阶段、事件冒泡阶段。

首先发生的是事件捕获，然后是实际的目标接收事件，最后是事件冒泡，可以在这个阶段对事件作出响应。

bind()和on(),live()和bind()的区别

bind直接绑定在一个对象上

这个例子的是绑定在 #foo 这个元素上。点击#foo元素后执行回调函数。

on方法是一个事件委托

这个例子是绑定在document上，点击document下的含有类offsite的a标签元才执行回调函数。

live方法是on方法的一种实现

$("a.offsite").live("click", function(){ 
    alert("Goodbye!"); 
});

$(document).on("click", "a.offsite", function(){ 
    alert("Goodbye!"); 
});

live方法是on方法的一种实现，上面这两个方法完全相同，live 方法是on方法的一种实现。

js中的类型识别

typeof

typeof是一种操作符，我们可以通过typeof(“a”)或者直接typeof “a”来调用。注意：typeof并不是一种方法。

typeof 总结：

  • 可以识别标准类型（Null除外）
  • 不能识别具体的对象类型（Function 除外）

instanceof

instanceof用来识别内置的对象类型,不能够识别原始类型。

instanceof 能够用来判别自定义对象类型及父子类型

instanceof 总结：

  • 判别内置对象类型
  • 不能判别原始类型
  • 判别自定义对象类型

Object.prototype.toString.call

Object.prototype.toString.call总结：

  • 可以识别标准类型以及内置（build-in）对象类型
  • 不能识别自定义对象类型

constructor

constructor就是构造这个对象的构造函数本身。

我们用构造函数去识别这个对象是什么类型肯定是准确的。

constructor 总结：

  • 判断标准类型（Undefined/Null 除外，因为前两者没有构造函数）
  • 判别内置对象类型
  • 判别自定义对象类型
